1. Porting Support Tool Overview

1-1. Overview

This Porting Support Tool (hereinafter called "Tool") converts the C-language program file of Pocket Viewer for LSIC (C compiler made by LSI Japan, Co., Ltd.) into C-language program file for Hitachi C complier. This Tool converts for C-language only, not to Assembly Language. Depending on program structure, it may be necessary to modify by user.

Return

3. Porting Conversion Items (LSIC)

3-1. FAR Declaration Conversion

(Difference between LSIC and Hitachi C compiler)

FAR is a function that is not included in standard C-language and it is a unique function to i8086 (LSIC). i8086 has two kind of access method in addoressing area, (space that accepts accesses efficiently though it is small) and (space that is costly to access though it is large). LSIC uses FAR as a keyword to put in the large space. Hitachi C has no such concept.
 

(Contents of conversion)

The place where FAR declaration is set is searched and conversion is done. Conversion methods available have two; one is FAR declaration simple delete and the other is FAR declaration delete + CONST declaration add.

(Conversion example)

'FAR simple delete'

Example 1) unsigned char far p;

            v

unsigned char p;

Example 2) work = ( char far * )Data[1];

            v

work = ( char * )Data[1];

'FAR delete + CONST definition add'

Example 1) int far a = 1;

            v

int const a = 1;

Example 2) char byte far data[ ] = {"1234"};

            v

char const data [ ] = {"1234"};

Return

(Difference between LSIC and Hitachi C compiler)

Hitachi C compiler has no concept to use far so that FAR Processing Function does not exist.

(Contents of conversion)

Converts FAR Processing Function to normal string function.

(Conversion example)

Example 1) far_memcpy();

            v

     memcpy();

Example 2) far_memmove();

            v

memmove();

3-3 NULL Conversion

(Difference between LSIC and Hitachi C compiler)

Difference exists in NULL definition.

    LSIC  :    #define NULL 0

    Hitachi C    :    #define NULL (( void * ) 0 )

(Contents of conversion)

To match with NULL definition of LSIC, the value used for NULL is converted to zero - [0].

(Conversion example)

int *p = NULL;

            v

int *p = 0;

Return

3-6. Macro Declaration Conversion

(Difference between LSIC and Hitachi C compiler)

Difference exists in format that has argument to remove Macro Declaration.

LSIC:    #define identifier(identifier-list) token-sequence

            #undef identifier(identifier-list)

Hitachi C :    #define identifier(identifier-list) token-sequence

            #undef identifier

(Contents of conversion)

Convert Macro Declaration that has argument to remove into format of Hitachi compiler.

(Conversion example)

#define MAX( A, B )    ((A) > (B) ? (A) : (B))

#undef  MAX( A, B )

             v

#define MAX( A, B )    ((A) > (B) ? (A) : (B))

#undef  MAX

Return

Convert compiler control command [#endif]that used LABEL into format of Hitachi compiler.

Delete empty sentence [;]described outside declaration of function.

(Conversion example)

{
};

            v

void sbrk( int x)
{
}

4. Caution of Porting Tool (LSIC)

4-1. Warning to Mismatched Type Declaration

As error detecting level differs between LSIC and Hitachi C compiler, the warning that was not detected on LSIC may appear. Porting Tool will operate even if the corrections at the following conditions are not done. Any warning that occurs is not converted in Porting Tool.

(Contents of warning 1)

C1016 (W) Argument mismatch
    Argument in prototype declaration and the argument that corresponds to a call function are of pointer type; each pointer points different type. Pointer of argument in a call to function is set with internal representation as is.

（Warning example）

 1:     void test(void);
 2:     void sample( unsigned char * point, char data );
 3:     void test(void)                                    --------------
 4:     {
 5:           char work1[1];
 6:            char work2 = 1;                              Internal representation char
 7:            work1[0] = work2;
 8:            sample ( work1, work2 );
 9:     }                                                  --------------
10:    void sample ( unsigned char * point, char data )   --------------
11:    {                                                     Internal representation
12:             *point = data;                             unsigned char
13:    }                                                   --------------

*8th line becomes warning output.

(Method of correction)

Do cast conversion and match the type.

 8:            sample ( ( unsigned char * ) work1, work2 );

(Contents of warning 2)

C1000 (W) Illegal pointer assignment
    Pointer type substitutes each other. Thus, the type assigned by each pointer is different.
    Declaration at substituted side is set as is.
C1002 (W) Illegal pointer for "Operator"
    Binary operator (= =, !=, >, < and >=) or non-operator is not designating the pointer to the same type.

（Warning example）

 1:    void test(void);
 2:    void test(void)
 3:    {
 4:            unsigned char * work1;
 5:           char * work2;
 6:           work2 = work1;
 7:    }

*6th line becomes warning output.

(Method of correction)

Do cast conversion and match the type.

 6:            work2 = ( char * )work1;

(Contents of warning 3)
C1017 (W) Return type mismatch
    Type that is returned by function and type of return sentence are of pointer type; each pointer points different type.
    Internal representation of pointer in return sentence is set as is.

（Warning example）

1:     void test(void);
2:     char * sample( void );
3:     void test(void)
4:     {                                        --------------
5:             char＊point ;                      Internal representation
6:             point = sample( );                 char
7:     }                                        --------------          
8：    char * sample( void )
9:     {                                        --------------
10:            unsigned char*point;               Internal representation
11:            return point ;                   unsigned char
12:    }                                        --------------

Return

4-2. Handling of Floating Decimal Point

LSIC handles the floating decimal point constants as [long double type]. As Hitachi C compiler handles the values as [double type], the floating decimal point constants are up to 17 digits. In case that the values exceed 17 digits, warning described below will appear. When no correction for warning is done, the compiler would compute the values with reduced digits so that accuracy may get lower. The warning that occurs with respect to this description will not be converted in Porting Tool.

(Contents of warning)

C1024 (W) Out of float
    Valid digits of floating decimal point constants are exceeding 17 digits.

(Method of correction)

Make floating decimal point constants that exceed 17 digits to 17 digits.

(Example)
#define    PI_2    3.14159265358979323846264338327950288

                        v

#define    PI_2    3.1415926535897932

Return

4-4. Initialization of Recursive Call in Structure

Hitachi C compiler cannot designate variable that is not constant expression to default value of variable for structure type, Union type and array type and it provides error. The error that occurs with respect to this description will not be converted in Porting Tool.

(Contents of error)

C2130 (E) Initializer should be constant
    Compiler designates the variable that is not constant expression to default value of variable for structure type, union type and array type. It also designates the variable that is not constant expression to default value of variable that is statically allocated.

(Warning example)

 1: void main( void ){
 2: typedef struct a {
 3:   int i;
 4:   struct a *n;
 5: } AS;
 6: AS p[ ]={
 7:   { 1, &p[1] },
 8:   { 2, (void*) 0 }
 9: };
10: }

 *7th line becomes warning output.

(Method of correction 1)

As shown below, error should be prevented in a way that local variable is defined as static variable for enabling it to have fixed address. However, note that the variable is not created on stack and it acquires memory region.

 1: void main( void ){
 2: typedef struct a {
 3:      int i;
 4:      struct a *n;
 5:      } AS;
 6:    static AS p[ ]={  --Correction
 7:     { 1, &p[1] },
 8:     { 2, (void*) 0 }
 9: };
10:  }

(Method of correction 2)

As shown below, error is avoided if acquisition of variable region and setting of default value are not done simultaneously. In addition, the variable is created on stack.

 1: void main( void ){
 2: typedef struct a {
 3:     int i;
 4:     struct a *n;
 5:      } AS;
 6:    AS p[2];           /* Acquisition of variable region */
 7：     p[0].i = 1;  /* Initialization */
 8：     p[0].n = &p[1] ;
 9：     p[1].i = 2;
10:     p[1]．n = (void*) 0;
11: }

Return

4-5. Cast Conversion Insertion Position

Hitachi C does not covert cast at the head of left part value. In case that the left part is used, the following warning would appear. The error that occurs with respect to this description will not be converted in Porting Tool.
 

(Contents of warning)

C1022 (W) First operand of "Operator" is not value
No.1 "Operator" of operand is not left part value.
 

(Warning example)

1: unsigned int data;
2: (int) data -= -10;
3: if( data < 0 )
4: {
5: }
＊2nd line becomes warning output.
 

(Method of correction)

Add cast conversion to the place where code was assigned or not assigned.
1: unsigned int data;
2: data -= -10;
3: if( (int) data < 0 )
4: {
5: }

Return

4-6. Initialization of String

Hitachi C cannot use single and double quotation simultaneously when designating default value of variable. When used, error is provided. The error that occurs with respect to this description will not be converted in Porting Tool.
 

(Contents of error)

C2129 (E) Illegal initializer type
The specified default value for variable is not the type that can be substituted.
 
 

(Error example)

1: char mess[ ] = { ' ',0xf6,0xa1,'/','4',0xf6,0xb0,"R" };

(Method of correction)

1: char mess[ ] = { " /x06/xa1\/4/xf6/xb0R" };

Return

4-7.FP_OFF(),FP_SEG(),MK_FP()
- farPointer Segmentation/Composition

(Difference between LSIC and Hitachi compiler)

farPointer used on LSIC is comprised with 16-bit segment+16-bit offset - total of 32-bit. (Segment takes higher-level and offset takes lower-level). farPointer uses FP_OFF( ), FP_SEG( ), MK_FP( ) for segmentation and Composition. Hitachi C pointer has no concept of segment or offset so that FP_SEG( ), MK_FP( ) are not existing.

FP_OFF( ) : Offset acquisition
Acquires offset value of farPointer and returns it as unsigned type.
FP_SEG( ) : Segment acquisition
Acquires segment value of far Pointer and returns it as unsigned type.
MK_FP( ) : Address creation
farPointer will be created from segment and offset value of unsigned type.

(Contents of conversion)

When substituted with Hitachi C, the concept of pointer takes the policy same as LSIC. Thus, LSIC program used for pointer segmentation and composition will not be converted to correspond to Hitachi C. To attain the above, FP_OFF ( ), FP_SEG ( ) and MK_FP ( ) that are equivalent in processing operation to LSIC were macro-defined and added.

(Addition of Macro Definition)

#define  FP_SEG( p )  ( ( unsigned short ) ( ( unsigned long ) ( ( void * ) ( p ) )  >> 16 ) )
#define  FP_OFF( p )  ( unsigned short ) ( p )
#define  MK_FP( s, o )  ( ( void * ) ( ( ( unsigned long )( s ) << 16 ) | ( unsigned short ) ( o ) ) )

(Supplement)

(Difference between 8086-family CPU and Hitachi CPU Address)
8086-family CPU uses addressing method called segment system and uses dedicated register called segment register. Hitachi CPU has not segment register and it can be accessed to address space up to 4G byte (32-bit: [SH]).

Segment system:
It is created to combine compatibility support of 16-bit addressing (64K bytes address space) and extension (1M bytes). However, the address space that can be accessed directly is limited to 64K bytes. To access the space beyond, the dedicated register called segment register is used.
